React JS Machine Test – Real Estate Web 
Platform (MVP) 
Objective 
Build a responsive property listing web application using React JS that demonstrates 
search, listing, user roles, and basic interactions as per the MVP requirements. 
Tech Stack (Mandatory) 
● React JS (Functional Components) 
● React Router 
● Context API or Redux (any one) 
● CSS / MUI / Bootstrap (any one) 
● Mock APIs (JSON / local state / dummy data) 
● Google OAuth – mocked flow acceptable 
Core Requirements 
1. Search & Navigation 
Features 
● Location Search: 
○ Search properties by City / Locality / State 
● Filters: 
○ Property Configuration (1BHK, 2BHK, 3BHK) 
○ Budget Range (Min–Max) 
○ Possession Period (Ready / 6 months / 1 year) 
● Map View (Mock): 
○ Display properties on a map (Google Maps iframe OR placeholder component) 
Expected Screens 
● Home Page with: 
○ Search bar 
○ Filters 
○ Property cards grid 
○ Toggle between List View / Map View 
2. Property Listings 
Property Card 
Each card should display: 
● Property Name 
● Price 
● Location 
● Configuration 
● “View Details” button 
Property Details Page 
Route: /property/:id 
Must show: 
● Price 
● Full Location 
● Amenities list 
● Possession Date 
● Sample Flat Video (embedded / placeholder) 
● Building & Locality Video (embedded / placeholder) 
3. User Profiles 
Buyer 
● Can browse without login 
● Must login (Google / mock login) to: 
○ Save favorite properties 
○ Schedule video call / site visit 
● Buyer Dashboard: 
○ Saved properties 
○ Scheduled appointments 
Seller (Builder) 
● Registration required (mock payment checkbox) 
● Seller Dashboard: 
○ Add new property (form) 
○ Upload videos (file input / URL) 
○ View buyer inquiries 
○ View scheduled appointments 
4. Communication Tools (Mocked) 
● 
 
 
 Phone Call: 
○ Clickable phone number (tel: link) 
● 
 
 
 Video Call Scheduling: 
○ Date & time picker 
○ Confirmation screen 
● 
 
 
 Site Visit Scheduling: 
○ Date & time picker 
5. Admin Panel (Basic) 
Route: /admin 
Features 
● View all property listings 
● Approve / Reject property listings 
● View registered buyers and sellers 
● View all scheduled appointments 
6. Notifications (Mock) 
Trigger notifications on: 
● Appointment scheduled 
● Appointment approved 
Implementation: 
● Toast messages / alerts 
● Bonus: Email/SMS toggle UI (no backend needed) 
7. Authentication & Security 
● Google Login (mocked allowed) 
● Protected Routes: 
○ Buyer Dashboard 
○ Seller Dashboard 
○ Admin Panel 
● SSL – assumed (mention in README) 
8. Monetization (UI Level) 
● Seller Registration Fee: - Checkbox or mock payment screen 
● Premium Listing Option: - Toggle while adding property - Premium badge on property cards
